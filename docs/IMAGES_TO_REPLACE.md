## Especificaciones Técnicas

### Formatos Recomendados
- **Fotos generales**: JPG (calidad 85%)
- **Logos e iconos**: PNG con transparencia o SVG
- **Iconos PWA**: PNG
- **Favicon**: ICO o PNG

### Optimización
- Todas las imágenes deben estar optimizadas para web
- Usar herramientas como TinyPNG, ImageOptim o Squoosh
- Considerar formato WebP para mejor compresión
- Next.js optimizará automáticamente las imágenes servidas

### Nombres de Archivo
Usar nombres descriptivos en kebab-case:
- ✅ `golden-retriever-bano-antes.jpg`
- ✅ `jose-antonio-camacho-perfil.jpg`
- ❌ `IMG_1234.jpg`
- ❌ `foto perro.jpg`

### Derechos de Imagen
- Asegurarse de tener derechos sobre todas las imágenes
- Si se usan fotos de clientes, obtener consentimiento por escrito
- Considerar contratar un fotógrafo profesional para fotos clave
- Alternativa: usar servicios de stock con licencia comercial

## Proceso de Reemplazo

1. **Recopilar fotos reales** del negocio
2. **Optimizar** todas las imágenes
3. **Subir a Sanity** las imágenes de contenido (servicios, blog, galería, testimonios)
4. **Reemplazar en /public/** las imágenes estáticas (logo, favicon, OG image, PWA icons)
5. **Verificar** que todas las imágenes se muestren correctamente
6. **Probar** en diferentes dispositivos y tamaños de pantalla
7. **Optimizar** el rendimiento con Lighthouse

---

## Checklist de Verificación

Después de reemplazar las imágenes:

- [ ] Todas las imágenes se cargan correctamente
- [ ] No hay imágenes rotas (404)
- [ ] Las imágenes tienen alt text descriptivo
- [ ] Las dimensiones son apropiadas para cada uso
- [ ] Las imágenes están optimizadas (tamaño de archivo razonable)
- [ ] El sitio pasa Lighthouse con buen score de Performance
- [ ] Las imágenes se ven bien en móvil, tablet y escritorio
- [ ] Los iconos PWA funcionan correctamente
- [ ] La imagen OG se muestra bien al compartir en redes sociales
